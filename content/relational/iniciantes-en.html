Relational Database Fundamentals

From tables to basic queries. Master the essential commands: SELECT, WHERE, ORDER BY, and DISTINCT.

1. What is a Relational Database?

Based on relational algebra, this type of database organizes data into tables (relations). Each table has a unique name and consists of:

  • Columns (Fields): Vertical structure defining the data type (e.g., Name, Birth Date, Salary).
  • Rows (Records): Horizontal structure containing the actual data of an entity.

The main advantage is integrity: you cannot insert text into a column defined to store dates. This ensures data quality for future analysis.

2. Retrieving Data (SELECT)

The de>SELECT command is for reading. It does not alter data, it only presents it.

Selecting Specific Columns

It is good practice to list only the columns you need, rather than fetching the entire table.

de>SELECT NAME, JOB FROM PERSONS;

Selecting All (*)

The asterisk retrieves all available columns.

de>SELECT * FROM COUNTRIES;

3. Filtering Records (WHERE)

The de>WHERE clause restricts the returned rows. The database checks the condition for each row; if True, the row appears.

Data TypeHow to use in WHEREExample
NumbersDirectly, no quotes.de>WHERE POP > 1000000
Text (Strings)Inside single quotes.de>WHERE COUNTRY = 'Brazil'
DatesInside single quotes (YYYY-MM-DD).de>WHERE BDATE > '1990-01-01'

4. Sorting Results (ORDER BY)

By default, row order is not guaranteed. To organize output, use de>ORDER BY. It must always be the last clause.

  • ASC (Ascending): Default. Smallest to largest (A-Z, 0-9).
  • DESC (Descending): Largest to smallest (Z-A, 9-0).
de>SELECT COUNTRY, POP FROM COUNTRIES ORDER BY POP DESC;
