Fundamentos de Banco de Dados Relacional e SQL

Uma introdução completa sobre como dados são organizados em tabelas e como realizar suas primeiras consultas usando a linguagem SQL.

1. O que é um Banco de Dados Relacional?

Imagine um banco de dados relacional como uma coleção organizada de tabelas que se relacionam entre si. Diferente de uma planilha solta, aqui a estrutura é rígida e previsível, garantindo que os dados sejam consistentes.

A estrutura básica é composta por:

  • Tabelas (Tables): Onde os dados vivem. Pense nelas como "pastas" para categorias específicas de informações (ex: Clientes, Pedidos).
  • Colunas (Columns): Definem os atributos que armazenamos. Por exemplo, numa tabela de Pessoas, teríamos colunas para Nome, Idade e Cidade.
  • Linhas (Rows): Cada linha é um registro único. Se a tabela é Pessoas, uma linha representa uma pessoa específica.

2. Sua Primeira Consulta: O Comando SELECT

A linguagem SQL (Structured Query Language) é o padrão para "conversar" com esses bancos. O verbo mais importante que você vai aprender é o de>SELECT.

Ele serve para ler dados. Uma consulta SQL básica tem duas partes obrigatórias:

  1. SELECT: O que você quer ver? (Quais colunas?)
  2. FROM: De onde você quer pegar? (Qual tabela?)

Sintaxe Básica

de>SELECT coluna1, coluna2
  FROM tabela;

Se você quiser ver todas as colunas de uma tabela, usamos o símbolo asterisco (de>*). Cuidado: em bancos de dados reais com milhões de linhas, isso pode ser lento! Mas para aprender, é excelente.

Exemplo Prático

Para ver todos os dados da tabela de Países:

de>SELECT * FROM COUNTRIES;

Para ver apenas o nome e a população:

de>SELECT COUNTRY, POP FROM COUNTRIES;

3. Filtrando Resultados com WHERE

Raramente queremos ver o banco inteiro. Geralmente, precisamos de respostas específicas, como "Quais países falam Inglês?" ou "Quem nasceu antes de 1990?". É aqui que entra a cláusula de>WHERE.

O de>WHERE funciona como um filtro. Ele olha linha por linha e decide: "Esta linha atende à condição?". Se sim, ela aparece no resultado.

aption>Operadores Comuns no WHERE
Operador Significado Exemplo
de>= Igual a de>WHERE COUNTRY = 'USA'
de>> Maior que de>WHERE POP > 1000000
de>< Menor que de>WHERE AREA < 500
Dica Importante: Textos em SQL devem estar sempre entre aspas simples (de>'Texto'). Números não precisam de aspas.
