Fundamentos de Banco de Dados Relacional e SQL

Do conceito de tabelas até a ordenação de resultados. Domine os comandos essenciais SELECT, WHERE, ORDER BY e DISTINCT.

1. O que é um Banco de Dados Relacional?

Baseado no conceito de álgebra relacional, este tipo de banco organiza dados em tabelas (relações). Cada tabela possui um nome único e é composta por:

  • Colunas (Campos): Estrutura vertical que define o tipo de dado (ex: Nome, Data de Nascimento, Salário).
  • Linhas (Registros): Estrutura horizontal que contém os dados reais de uma entidade.

A grande vantagem é a integridade: você não pode inserir um texto numa coluna definida para guardar datas. Isso garante a qualidade da informação para análise futura.

2. Recuperando Dados (SELECT)

O comando de>SELECT é a ferramenta de leitura. Ele não altera os dados, apenas os apresenta.

Selecionando Colunas Específicas

É uma boa prática listar apenas as colunas que você precisa, ao invés de trazer a tabela toda.

de>SELECT NAME, JOB FROM PERSONS;

Selecionando Tudo (*)

O asterisco traz todas as colunas disponíveis.

de>SELECT * FROM COUNTRIES;

3. Filtrando Registros (WHERE)

A cláusula de>WHERE restringe as linhas retornadas. O banco de dados verifica a condição para cada linha; se for Verdadeira (True), a linha aparece.

Tipo de Dado Como usar no WHERE Exemplo
Números Direto, sem aspas. de>WHERE POP > 1000000
Texto (Strings) Entre aspas simples. de>WHERE COUNTRY = 'Brazil'
Datas Entre aspas simples (formato AAAA-MM-DD). de>WHERE BDATE > '1990-01-01'

4. Ordenando Resultados (ORDER BY)

Por padrão, a ordem das linhas em um banco de dados não é garantida. Para organizar a saída, usamos de>ORDER BY. Esta deve ser sempre a última cláusula da sua consulta.

  • ASC (Ascendente): Padrão. Do menor para o maior (A-Z, 0-9).
  • DESC (Descendente): Do maior para o menor (Z-A, 9-0).

Exemplo: Listar países do maior para o menor em população.

de>SELECT COUNTRY, POP 
  FROM COUNTRIES 
  ORDER BY POP DESC;

5. Eliminando Duplicatas (DISTINCT)

Muitas vezes, uma coluna tem valores repetidos (ex: vários clientes moram no mesmo "País"). Para saber apenas quais países únicos existem na tabela, usamos de>DISTINCT logo após o SELECT. 

de>SELECT DISTINCT COUNTRY FROM PERSONS;

Isso retornará cada país apenas uma vez, independente de quantas pessoas morem lá.

6. Apelidos de Coluna (Aliases)

Você pode renomear colunas no resultado para torná-las mais legíveis, usando a palavra-chave de>AS. Isso não muda o nome no banco, apenas na exibição. 

de>SELECT NAME AS Nome_Completo, BDATE AS Data_Nascimento 
  FROM PERSONS;
