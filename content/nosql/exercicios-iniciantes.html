Exercícios Práticos: Modelagem NoSQL

Bancos NoSQL orientados a documentos usam formatos como JSON. Vamos praticar a "tradução" de uma Tabela Relacional para um Documento JSON.

Desafio 1: Estruturando um Documento

Conceitual

Observe a linha abaixo de uma tabela relacional de>USERS:

IDNomeSkills
1AnaSQL, Python

No modelo relacional, teríamos problemas com o campo "Skills" (vários valores). No NoSQL, usamos um Array. Escreva abaixo como ficaria esse objeto em formato JSON (Chave: Valor).

Desafio 2: Schema Dinâmico

Conceitual

Imagine que você tem um cadastro de produtos. Alguns produtos têm "Voltagem" (Eletrônicos), outros têm "Tamanho" (Roupas).

Em SQL, você teria muitas colunas com de>NULL. Em NoSQL, cada documento tem apenas os campos que precisa.

Tarefa: Crie mentalmente dois objetos JSON representando uma "Camiseta" e uma "Geladeira". Note que eles não precisam ter as mesmas chaves.
