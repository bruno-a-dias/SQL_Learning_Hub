A Evolução do Armazenamento

1. RDBMS

Anos 80-90

Um servidor central. Tabelas rígidas.

2. Hadoop/NoSQL

Anos 2000-10

Clusters commodity. Esquema flexível.

3. Cloud Native

Hoje

Separação de Compute & Storage. (Snowflake, BigQuery).

Lake vs Warehouse

Característica Data Warehouse Data Lake
Dados Estruturados, Limpos Brutos (Raw), Variados
Custo Alto (Armazenamento rápido) Baixo (Object Storage S3)
Usuário Analista de BI Cientista de Dados

Arquitetura "Lakehouse"

O novo padrão da indústria:

  • Armazene TUDO no Lake (barato).
  • Use Engines SQL (Athena, Presto) para consultar direto do Lake.
  • Mova para o Warehouse apenas os dados "Ouro" (KPIs críticos).
