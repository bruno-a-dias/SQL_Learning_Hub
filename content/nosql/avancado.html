Arquitetura de Big Data: Lakes vs Warehouses

Como as grandes empresas organizam petabytes de dados? A convergência entre o mundo relacional e o não relacional na nuvem.

1. Data Warehouse (Armazém de Dados)

É um banco de dados relacional gigante, otimizado para leitura (Analytics).

  • Estrutura: Altamente estruturado (SQL). Usa esquema "Estrela" ou "Snowflake".
  • Processo: ETL (Extract, Transform, Load). Você limpa o dado antes de guardar.
  • Exemplos: Amazon Redshift, Snowflake, Google BigQuery (modo nativo).

2. Data Lake (Lago de Dados)

Um repositório centralizado que permite armazenar todos os dados estruturados e não estruturados em qualquer escala.

  • Estrutura: Sem estrutura definida. Aceita CSV, JSON, Imagens, Logs, Parquet.
  • Processo: ELT (Extract, Load, Transform). Você guarda tudo bruto e limpa apenas quando for usar.
  • Tecnologia: Hadoop (HDFS), Amazon S3, Azure Data Lake.

3. O Melhor dos Dois Mundos: SQL Engines

Hoje, não precisamos escolher. Ferramentas modernas permitem escrever SQL padrão para consultar arquivos JSON ou CSV jogados num Data Lake, sem precisar importar para um banco.

Isso é mencionado no material como "Cloud-Based SQL Platforms" (Capítulo 30). Ferramentas como AWS Athena ou Presto leem arquivos onde eles estão, aplicam esquema na leitura (Schema-on-Read) e retornam tabelas.

Conceito Chave: Separar Computação (Query Engine) do Armazenamento (Data Lake). Isso permite escalar cada um independentemente.
